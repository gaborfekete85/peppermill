## Scope

Smallest end-to-end agent that runs.

**In:**
- `bus.events`: `InboundMessage`, `OutboundMessage` (dataclasses).
- `bus.queue`: `MessageBus` (two `asyncio.Queue`s + publish/consume helpers).
- `providers.fake`: `FakeProvider`, `LLMResponse`, `ToolCallRequest`
  (concrete classes, no ABC yet).
- `agent.tools.add`: `AddTool` (concrete class with `schema()` + async
  `execute(a, b)`).
- `agent.loop`: `AgentLoop` with two methods — `run_once(msg)` and
  `run_forever()`. Linear loop, no FSM, no sessions.
- `channels.cli`: `CLIChannel` (stdin reader + stdout writer using
  `run_in_executor` for blocking `sys.stdin.readline`).
- `__main__`: wires everything; preloads a scripted FakeProvider.

**Out (deferred to later versions):**
- LLMProvider ABC (v0.2)
- TurnState FSM, AgentRunner extraction (v0.3)
- Sessions, persistence, history (v0.4)
- Tool registry, more tools, workspace restriction (v0.5)
- ContextBuilder, Jinja2 templates (v0.6)
- Streaming (v0.7)
- Context governance (v0.8)
- Crash recovery (v0.9)
- Concurrency primitives (v0.10)
- BaseChannel ABC, second channel (v0.11)
- Second provider, factory (v0.12)
- Config (v0.13)
- Web tool + SSRF (v0.14)

## Specific design decisions

1. **FakeProvider takes `script: list[LLMResponse]`**, not a generator
   function. Easier to test, harder to misuse (no infinite scripts).
   Exhausting the script raises `RuntimeError`.
2. **AgentLoop hardcodes the tool dict in `__main__`** — no registry yet.
3. **`run_in_executor(None, sys.stdin.readline)`** for stdin. We
   deliberately learn this pattern early because every "block in a thread"
   case in later versions will look like this.
4. **CLIChannel is concrete, not an ABC subclass**, but shaped so v0.11
   can retrofit `BaseChannel` cleanly.
5. **No graceful shutdown beyond `KeyboardInterrupt`.** v0.10 introduces
   proper task lifecycle.

## Done when

- `python -m peppermill` runs, accepts input, returns `"The answer is 5."`.
- All unit tests pass: `pytest -v` is green.
- E2E test (`tests/test_e2e.py`) drives the full pipeline through the bus.
- `ruff check peppermill/ tests/` is clean.
- `docs/architecture.md` exists and describes v0.1.
- PR opened from `v0.1` → `main`.

## Test plan summary

- `tests/bus/test_events.py` — dataclass construction, `session_key`
  derivation, default factories.
- `tests/bus/test_queue.py` — publish/consume roundtrip on both queues,
  queue independence.
- `tests/providers/test_fake.py` — scripted-sequence behaviour, exhaustion
  raises.
- `tests/agent/tools/test_add.py` — `execute` correctness, `schema` shape.
- `tests/agent/test_loop.py` — `run_once` with tool call, `run_once` with
  direct text, unknown-tool soft failure.
- `tests/test_e2e.py` — drives `run_forever` through the bus end-to-end
  with a 2-second timeout safeguard.
