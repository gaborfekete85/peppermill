# v0.4 Design: Sessions + Persistence

**Date:** 2026-06-04  
**Author:** Claude (brainstorming + design)  
**Status:** Approved by user  

---

## Overview

v0.4 adds session memory to peppermill. Conversations survive across restarts by persisting turn summaries to disk. The agent can load past conversation history and use it as context for new turns, enabling stateful multi-turn conversations.

**Key principle:** Conversations are identified by session key (derived from `InboundMessage.session_key`, defaults to `channel:chat_id`). Each session has its own history file on disk.

---

## FSM Extension

### Current (v0.3)
```
BUILD → RUN → RESPOND → DONE
```

### New (v0.4)
```
RESTORE → BUILD → RUN → SAVE → RESPOND → DONE
```

**State handlers:**

| State | Responsibility | Input | Output |
|-------|---|---|---|
| **RESTORE** | Load past session history from disk into context | `ctx.inbound` (contains session_key) | `ctx.history_records` (list of TurnSummary), `ctx.messages` (seeded from history) |
| **BUILD** | Append new inbound message to message context | `ctx.inbound`, `ctx.messages` | `ctx.messages` updated with new user message, `ctx.tool_schemas` built |
| **RUN** | Delegate to AgentRunner (unchanged from v0.3) | `ctx.messages`, `ctx.tool_schemas` | `ctx.result` (AgentRunResult) |
| **SAVE** | Persist the turn to history.jsonl | `ctx.result`, `ctx.history_records`, `ctx.inbound` | History file updated with new TurnSummary |
| **RESPOND** | Publish outbound message (unchanged from v0.3) | `ctx.result`, `ctx.inbound` | OutboundMessage published |
| **DONE** | Sentinel (unchanged from v0.3) | — | FSM exits |

### State Transition Logic

All states return the next state sequentially (linear path). No branching conditions in v0.4.

## Storage & Persistence

### Session Key

Derived from `InboundMessage.session_key`:
- **Default:** `f"{inbound.channel}:{inbound.chat_id}"` (e.g., `"cli:main"`, `"telegram:12345"`)
- **Override:** `inbound.session_key_override` if set (future: per-user sessions)

### History File Location

```
~/.peppermill/sessions/{session_key}/history.jsonl
```

Example:
```
~/.peppermill/sessions/cli:main/history.jsonl
~/.peppermill/sessions/telegram:12345/history.jsonl
```

**Rationale:** `~/.peppermill/` is the standard location (mirrors nanobot). Session-specific dirs keep conversations isolated. Configurable in v0.13 when Config is added.

### File Format: JSONL (JSON Lines)

Each line is a **TurnSummary** JSON object, one per turn:

```json
{"turn": 1, "timestamp": "2026-06-04T10:30:00Z", "user_message": "What is 2+3?", "assistant_response": "The answer is 5.", "tools_used": ["add"], "stop_reason": "completed"}
{"turn": 2, "timestamp": "2026-06-04T10:30:15Z", "user_message": "And 3+4?", "assistant_response": "The answer is 7.", "tools_used": ["add"], "stop_reason": "completed"}
```

### TurnSummary Dataclass

```python
@dataclass
class TurnSummary:
    turn: int                      # 1-indexed turn number in this session
    timestamp: str                 # ISO 8601 (e.g., "2026-06-04T10:30:00Z")
    user_message: str              # The raw user input
    assistant_response: str        # The final assistant text (or "" if None)
    tools_used: list[str]          # Tool names invoked (e.g., ["add", "add"])
    stop_reason: str               # "completed" | "max_iterations" | other
```

**Rationale for compact format:**
- Space-efficient (one entry per turn vs. one per message)
- Preserves essential conversation thread
- Stores which tools were used (without full arguments/results — those come in v0.9+ with audit logs)
- Sufficient to resume context in v0.4

### Write Strategy: Simple Append

```python
with open(history_file, 'a') as f:
    f.write(json.dumps(turn_summary_dict) + '\n')
```

**Trade-off:** Not atomic. If process crashes mid-write, a partial line may corrupt the file.  
**Mitigation for v0.4:** RESTORE skips malformed JSON lines (JSONDecodeError caught, logged, line skipped).  
**Future:** v0.9 upgrades to atomic writes (temp file + rename + fsync per nanobot).

## Components & Modules

### New Files

#### `peppermill/session/history.py`
**Purpose:** Low-level I/O for history.jsonl files.

**Functions:**
```python
def load_history(session_key: str, base_path: Path = None) -> list[dict]:
    """Load all turns from history.jsonl for a session.
    
    Args:
        session_key: e.g., "cli:main"
        base_path: defaults to ~/.peppermill/sessions/
        
    Returns:
        List of turn dicts (parsed JSON), or [] if file doesn't exist.
        Silently skips malformed JSON lines (logs warning).
    """

def append_turn(session_key: str, turn_dict: dict, base_path: Path = None) -> None:
    """Append one turn to history.jsonl.
    
    Args:
        session_key: e.g., "cli:main"
        turn_dict: TurnSummary as dict
        base_path: defaults to ~/.peppermill/sessions/
        
    Raises:
        OSError if directory creation fails.
    """

def ensure_session_dir(session_key: str, base_path: Path = None) -> Path:
    """Create session directory if it doesn't exist.
    
    Args:
        session_key: e.g., "cli:main"
        base_path: defaults to ~/.peppermill/sessions/
        
    Returns:
        Path to session directory.
        
    Raises:
        OSError if creation fails (permissions, disk full, etc.).
    """
```

#### `peppermill/session/manager.py`
**Purpose:** High-level session API, wraps history.py.

```python
class SessionManager:
    """Manages session history for one or more conversations."""
    
    def __init__(self, base_path: Path = None):
        """Initialize SessionManager.
        
        Args:
            base_path: Root for all sessions. Defaults to ~/.peppermill/sessions/.
        """
    
    async def load(self, session_key: str) -> list[dict]:
        """Load turn summaries for a session."""
    
    async def save(self, session_key: str, turn_summary: TurnSummary) -> None:
        """Append a turn to a session's history."""
```

**Design note:** Async methods prepare for v0.5+ when I/O might be concurrent. For v0.4, implementations can be simple `await`-wrapped sync code.

## Data Flow: One Turn in Detail (v0.4)

**Scenario:** User types `"What is 3+4?"` on second turn of a `cli:main` session.

1. **RESTORE** — `_state_restore`
   - Extract `ctx.session_key = inbound.session_key` → `"cli:main"`
   - Call `session_manager.load("cli:main")`
   - **First turn:** File doesn't exist → returns `[]`, `ctx.history_records = []`
   - **Second turn:** File exists with one turn → `ctx.history_records = [{"turn": 1, "timestamp": "...", ...}]`
   - Seed `ctx.messages` from history records (convert to assistant/user message format for LLM context)
   - Return `BUILD`

2. **BUILD** — `_state_build`
   - If history non-empty: `ctx.messages` already contains past context
   - Append new user message: `ctx.messages.append({"role": "user", "content": "What is 3+4?"})`
   - Build tool schemas: `ctx.tool_schemas = [add_tool.schema()]`
   - Return `RUN`

3. **RUN** — `_state_run`
   - Build `AgentRunSpec` with `ctx.messages` (which now includes past + new user message)
   - Call `self._runner.run(spec)`
   - Claude Haiku sees: `[{past context from turn 1}, {role: "user", content: "What is 3+4?"}]`
   - Returns `AgentRunResult(final_content="The answer is 7.", messages=[...], tools_used=["add"], ...)`
   - Store in `ctx.result`
   - Return `SAVE`

4. **SAVE** — `_state_save`
   - Extract from `ctx.result`:
     - `user_message = "What is 3+4?"`
     - `assistant_response = "The answer is 7."`
     - `tools_used = ["add"]`
     - `stop_reason = "completed"`
   - Build `TurnSummary(turn=2, timestamp="2026-06-04T...", user_message="...", ...)`
   - Call `session_manager.save("cli:main", turn_summary)`
   - File now has two lines (turn 1 + turn 2)
   - Return `RESPOND`

5. **RESPOND** — `_state_respond`
   - Publish `OutboundMessage(channel="cli", chat_id="main", content="The answer is 7.")`
   - Return `DONE`

6. **DONE**
   - FSM exits, `run_once` returns

**On third turn:** RESTORE loads both turns, history_records has 2 entries, messages are seeded with 2 past turns, new message appended, LLM sees full context.

## Error Handling

### RESTORE State

**Scenario:** History file corrupted (malformed JSON line)
- `load_history()` catches `json.JSONDecodeError`
- Log warning: `"Skipping malformed JSON line in session cli:main: ..."`
- Return list of successfully-parsed turns (skip the corrupted line)
- Conversation continues as if that line didn't exist
- **Data loss risk:** One turn is lost, but agent doesn't crash

**Scenario:** Session directory doesn't exist yet (first turn)
- `load_history()` catches `FileNotFoundError`
- Return empty list
- Conversation starts fresh (no error raised)

### SAVE State

**Scenario:** Directory creation fails (permissions denied, disk full)
- `ensure_session_dir()` raises `OSError`
- `_state_save` catches it, logs error: `"Failed to save session cli:main: ..."`
- Does NOT raise to user (conversation continues)
- **Data loss:** Turn is not persisted
- **Next restart:** Session history lacks this turn

**Scenario:** Write fails mid-line (disk full, process killed)
- Partial JSON written to file
- **On next RESTORE:** `load_history()` skips malformed line
- Conversation continues, missing one turn

### Summary

**Error philosophy for v0.4:**
- All errors are logged, not raised to user
- Conversation always completes (never crashes due to persistence)
- Graceful degradation: if save fails, continue without persisting
- **Future:** v0.9 adds atomic writes + crash recovery + proper durability

### Test Infrastructure

#### `InMemorySessionManager` (test double)
```python
class InMemorySessionManager:
    """Mock SessionManager for tests — stores in dict, not disk."""
    
    def __init__(self):
        self.sessions: dict[str, list[dict]] = {}  # session_key → turns
    
    async def load(self, session_key: str) -> list[dict]:
        return self.sessions.get(session_key, [])
    
    async def save(self, session_key: str, turn: dict) -> None:
        if session_key not in self.sessions:
            self.sessions[session_key] = []
        self.sessions[session_key].append(turn)
```

Placed in `tests/_helpers/in_memory_session_manager.py`.

### Hermetic Testing

All loop + runner tests use `InMemorySessionManager`. Real disk I/O only in `test_history.py` and `test_manager.py`, which use temp directories.

No real `~/.peppermill/sessions/` pollution.

## Backward Compatibility

**v0.3 code works as-is:**
- `AgentLoop.__init__` gains optional `session_manager` parameter (defaults to `SessionManager()` if not provided)
- Existing tests can inject `InMemorySessionManager` to avoid disk I/O
- `__main__.py` is updated to pass `session_manager`, but remains simple

**No breaking changes to public APIs.**

## Open Questions for v0.5+

1. **History format versioning:** If v0.5 changes TurnSummary fields, how do we migrate old files?
2. **TTL/cleanup:** When should old sessions be deleted? (v0.10)
3. **Compression:** History files could grow large over months. Gzip support? (v0.10)
4. **Full message audit:** Should we store complete message history separately for debugging? (v0.9)
5. **Per-user sessions:** Different users in the same chat_id. Override session_key? (v0.10)

## Success Criteria

- ✅ Conversations persist across restarts
- ✅ Session key defaults to `channel:chat_id` (overridable via InboundMessage)
- ✅ History stored as compact JSONL (turn summaries, not full messages)
- ✅ Simple append writes (no atomic durability in v0.4)
- ✅ Graceful error handling (all errors logged, none raised)
- ✅ Hermetic tests (no disk pollution)
- ✅ FSM extended cleanly (RESTORE and SAVE states fit linear progression)
- ✅ Backward compatible (v0.3 tests still pass)
