## 3. `LLMProvider` ABC

```python
from abc import ABC, abstractmethod

class LLMProvider(ABC):
    @abstractmethod
    async def chat(
        self,
        messages: list[dict[str, Any]],
        tools: list[dict[str, Any]] | None = None,
    ) -> LLMResponse: ...
```

Single abstract method for v0.2. v0.7 adds `chat_stream`; v0.2's ABC stays minimal.

`LLMResponse` and `ToolCallRequest` stay as plain `@dataclass`es with the same fields they had in v0.1 — only their location changes.

## 4. `AnthropicProvider` internals

### Construction

```python
AnthropicProvider(
    api_key: str | None = None,        # falls back to ANTHROPIC_API_KEY env var
    model: str = "claude-haiku-4-5-20251001",
    max_tokens: int = 4096,
    timeout: float = 60.0,
    base_url: str = "https://api.anthropic.com",
    client: httpx.AsyncClient | None = None,  # for tests; if set, base_url & timeout are ignored
)
```

If neither `api_key` arg nor `ANTHROPIC_API_KEY` env var is set, raise `RuntimeError("ANTHROPIC_API_KEY not set")` at construction time, **not** at first call. Fail loud, fail early.

When `client` is supplied (test injection via `httpx.MockTransport`), the constructor uses it as-is and ignores `base_url`/`timeout` — the caller owns those settings. When `client` is `None`, the constructor builds `httpx.AsyncClient(base_url=base_url, timeout=timeout)` itself.

### Per-call flow

```
chat(messages, tools)
  │
  ├── _build_request(messages, tools) ──► dict body
  ├── _client.post("/v1/messages", json=body, headers=...)
  ├── response.raise_for_status()                ← let httpx errors propagate
  └── _parse_response(response.json()) ──► LLMResponse
```

### `_build_request`

Body shape (Anthropic Messages API):

```python
{
  "model": self.model,
  "max_tokens": self.max_tokens,
  "messages": [...],          # translated from our internal format
  "tools": [...] | omitted,   # translated from our internal format
}
```

Headers:

```python
{
  "x-api-key": self.api_key,
  "anthropic-version": "2023-06-01",
  "content-type": "application/json",
}
```

Message translation rules:
- `{"role": "user", "content": str}` → unchanged.
- `{"role": "assistant", "content": str}` → unchanged (when no tool calls).
- `{"role": "assistant", "tool_calls": [{id, name, arguments}, ...]}` (our v0.1 shape from the AgentLoop) → `{"role": "assistant", "content": [{"type": "tool_use", "id": ..., "name": ..., "input": ...}, ...]}`.
- `{"role": "tool", "tool_call_id": id, "content": str}` (our v0.1 shape) → folded into a single `{"role": "user", "content": [{"type": "tool_result", "tool_use_id": id, "content": str}, ...]}` per assistant turn. v0.2 keeps it simple: one tool-result block per `tool` message, each in its own user turn. Compaction comes later if needed.

Tool translation: `schema()` returns `{name, description, parameters}` (OpenAI shape from v0.1). Anthropic wants `{name, description, input_schema}`. One-function rename:

```python
def _to_anthropic_tool(schema: dict) -> dict:
    return {
        "name": schema["name"],
        "description": schema["description"],
        "input_schema": schema["parameters"],
    }
```

### `_parse_response`

Anthropic returns:

```python
{
  "id": "msg_...",
  "type": "message",
  "role": "assistant",
  "content": [
    {"type": "text", "text": "..."},
    {"type": "tool_use", "id": "...", "name": "...", "input": {...}},
    ...
  ],
  "stop_reason": "end_turn" | "tool_use" | "max_tokens" | ...,
  "usage": {...},
}
```

Parsing:
- Concatenate all `text` blocks → `LLMResponse.content` (None if empty).
- Each `tool_use` block → `ToolCallRequest(id=block["id"], name=block["name"], arguments=block["input"])`.
- Map `stop_reason`: `"end_turn"` → `"stop"`, `"tool_use"` → `"tool_calls"`, `"max_tokens"` → `"length"`, anything else → keep as-is.

### Error handling (minimal)

- HTTP 4xx/5xx → `response.raise_for_status()` raises `httpx.HTTPStatusError`. Propagates out of `chat()`. AgentLoop catches it in `run_forever`'s `except Exception` and logs.
- Connection/timeout errors → propagate (`httpx.ConnectError`, `httpx.ReadTimeout`, etc.).
- Malformed response JSON → propagate (`json.JSONDecodeError` or `KeyError`).
- v0.2 does **not** retry. Retries are v0.7/v0.8's problem.

### Client lifecycle

Construct one `httpx.AsyncClient` in `__init__`, store as `self._client`. Add `async def aclose(self)` that closes it. `__main__.py` isn't shut down cleanly in v0.1 (Ctrl-D / Ctrl-C just exits the process), so we accept connection-pool warnings on shutdown for v0.2; v0.10 fixes lifecycle.

## 5. Test strategy

### `tests/providers/test_anthropic.py` — the new heavyweight

Uses `httpx.MockTransport` to inject canned responses. Pattern:

```python
def _client_with_response(payload: dict, status: int = 200) -> httpx.AsyncClient:
    def handler(request: httpx.Request) -> httpx.Response:
        return httpx.Response(status, json=payload)
    return httpx.AsyncClient(transport=httpx.MockTransport(handler))
```

Then a `AnthropicProvider` is constructed with that injected client (constructor accepts an optional `client` param for test injection — pedagogical "dependency injection" example).

Coverage:
- Request body shape (model, max_tokens, messages translated, tools translated).
- Headers (x-api-key, anthropic-version).
- Response parsing: text-only.
- Response parsing: tool_use blocks → ToolCallRequest.
- Response parsing: mixed text + tool_use.
- `stop_reason` mapping: end_turn → "stop", tool_use → "tool_calls", max_tokens → "length".
- Missing `ANTHROPIC_API_KEY` and no `api_key` arg → constructor raises.
- HTTP 401 → `chat()` raises `httpx.HTTPStatusError`.
- HTTP 500 → same.

### `tests/providers/test_base.py` — small but instructive

- Cannot instantiate `LLMProvider` directly (raises `TypeError`).
- A subclass that doesn't implement `chat` cannot be instantiated either.
- A complete subclass instantiates fine.

### `tests/agent/test_loop.py` — rewritten for ScriptedProvider

Same assertions as v0.1, only the import and constructor lines change. No new test cases added in v0.2.

### `tests/test_e2e.py` — rewritten for ScriptedProvider

Same as above. Real-API E2E intentionally deferred.

### `tests/_helpers/scripted_provider.py`

```python
class ScriptedProvider(LLMProvider):
    def __init__(self, script: list[LLMResponse]) -> None: ...
    async def chat(self, messages, tools=None) -> LLMResponse: ...
```

Lives under `tests/` so it's clearly not production code, but subclasses `LLMProvider` so the ABC contract is enforced.

## 6. Python concepts taught in v0.2

- **`abc.ABC` + `@abstractmethod`** — enforced subclass contract. Contrast with the `Protocol` already used in `loop.py`.
- **When to pick ABC vs Protocol** — short answer: ABC when you want explicit "is-a" + enforced inheritance; Protocol when you want duck-typing for "any object with these methods".
- **`httpx.AsyncClient`** — async HTTP, connection pooling, context-manager lifecycle.
- **`httpx.MockTransport`** — hermetic HTTP testing, no network bytes ever sent.
- **`os.environ.get(...)` + fail-loud constructor pattern** — secrets handling done correctly: detected at object creation, not buried in the first call site.
- **Dependency injection via constructor** — accepting `client: httpx.AsyncClient | None = None` so tests can inject a mocked client without monkey-patching.
- **Module-level refactor mechanics** — moving `LLMResponse`/`ToolCallRequest` from `fake.py` to `base.py` while keeping all consumers compiling.
