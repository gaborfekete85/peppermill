## 1. Scope

Promote `AgentLoop` from v0.2's linear `run_once` into a small **TurnState finite-state machine**, and **extract** the LLM-conversation work (provider calls + tool execution) into a separate `AgentRunner`. Net behaviour identical to v0.2; internals reorganized so future versions (sessions in v0.4, checkpoints in v0.9, etc.) have clean homes to land.

**In:**
- New file `peppermill/agent/runner.py` containing:
  - `AgentRunSpec` dataclass — runner input (initial messages, tool registry, provider, max_iterations).
  - `AgentRunResult` dataclass — runner output (final_content, full message history, tools_used, stop_reason).
  - `AgentRunner` class with a single `async def run(spec: AgentRunSpec) -> AgentRunResult` method.
- Rewritten `peppermill/agent/loop.py`:
  - `TurnState` enum (`BUILD`, `RUN`, `RESPOND`, `DONE`).
  - `_TurnContext` dataclass (mutable per-turn scratch space).
  - `AgentLoop.run_once(msg)` drives the FSM via `match/case`.
  - `run_forever()` unchanged in behaviour.
- Split tests:
  - `tests/agent/test_runner.py` — covers AgentRunner in isolation (uses `ScriptedProvider`).
  - `tests/agent/test_loop.py` — rewritten to cover the FSM at the orchestration level (uses a `ScriptedRunner` test helper).
- `tests/_helpers/scripted_runner.py` — new test-only stand-in for `AgentRunner`.

**Out (later versions, do NOT touch in v0.3):**
- `RESTORE`, `SAVE` states & their handlers (v0.4 — sessions).
- `COMPACT` state (v0.8 — context governance / auto-compact).
- `COMMAND` state (later — `/slash` command dispatch).
- Mid-turn message injection, checkpointing, length recovery (v0.7–v0.10).
- Streaming (v0.7).
- Crash-recovery checkpoints from runner (v0.9).
- Concurrency primitives, session locks (v0.10).

## 3. The FSM

```python
class TurnState(Enum):
    BUILD   = auto()    # assemble initial messages from inbound + tool schemas
    RUN     = auto()    # delegate to AgentRunner.run(spec); store result
    RESPOND = auto()    # publish OutboundMessage from runner result
    DONE    = auto()    # sentinel
```

### Routing (`run_once`)

```python
async def run_once(self, msg: InboundMessage) -> None:
    ctx = _TurnContext(inbound=msg)
    state = TurnState.BUILD
    while state is not TurnState.DONE:
        match state:
            case TurnState.BUILD:
                state = await self._state_build(ctx)
            case TurnState.RUN:
                state = await self._state_run(ctx)
            case TurnState.RESPOND:
                state = await self._state_respond(ctx)
            case _:
                raise RuntimeError(f"unknown state: {state}")
```

### State handlers

Each handler returns the next `TurnState`. No transition table — just direct returns. For 4 states this is clearer than indirection.

- `_state_build(ctx)`: seed `ctx.messages` with the user message + `ctx.tool_schemas` from the registry. → `RUN`.
- `_state_run(ctx)`: build `AgentRunSpec` from `ctx`, call `await self._runner.run(spec)`, store the `AgentRunResult` on `ctx.result`. → `RESPOND`.
- `_state_respond(ctx)`: build `OutboundMessage` from `ctx.result.final_content` + `ctx.inbound`, publish to bus. → `DONE`.

If a handler raises, the existing `except Exception` in `run_forever` catches and logs it. No new error handling.

### `_TurnContext`

```python
@dataclass
class _TurnContext:
    inbound: InboundMessage
    messages: list[dict[str, Any]] = field(default_factory=list)
    tool_schemas: list[dict[str, Any]] = field(default_factory=list)
    result: AgentRunResult | None = None
```

Mutable scratch space passed between handlers. Each handler reads/writes the fields it needs. Keeps handler signatures uniform: `(self, ctx) -> TurnState`.

## 4. AgentRunner

### Contract

```python
@dataclass
class AgentRunSpec:
    initial_messages: list[dict[str, Any]]
    tools: dict[str, ToolLike]
    tool_schemas: list[dict[str, Any]]
    provider: LLMProvider
    max_iterations: int = 20

@dataclass
class AgentRunResult:
    final_content: str | None
    messages: list[dict[str, Any]]          # complete history after the run
    tools_used: list[str]                   # names of tools invoked, in order
    stop_reason: str                        # "completed" | "max_iterations" | "error"

class AgentRunner:
    async def run(self, spec: AgentRunSpec) -> AgentRunResult: ...
```

### Behaviour

The `run` method's loop is exactly what v0.2's `AgentLoop.run_once` does inline, minus the outbound publish:

1. Start with `messages = list(spec.initial_messages)` and `tools_used: list[str] = []`.
2. For up to `spec.max_iterations` rounds:
   1. Call `await spec.provider.chat(messages, tools=spec.tool_schemas)`.
   2. If `response.has_tool_calls`:
      - Append the assistant tool-call message.
      - For each tool call, look up the tool in `spec.tools`, execute, append the tool-result message. Track `tools_used`.
      - Continue loop.
   3. If `response.has_tool_calls` is False:
      - Return `AgentRunResult(final_content=response.content, messages, tools_used, stop_reason="completed")`.
3. If the for-loop exhausts (hit `max_iterations`):
   - Return `AgentRunResult(final_content=None, messages, tools_used, stop_reason="max_iterations")`.

### Error handling

- Unknown tool → result string `f"error: unknown tool '{name}'"`, no crash. (Matches v0.2 behaviour.)
- Tool raises → result string `f"error: {exc}"`, no crash.
- Provider raises → propagate. AgentLoop's `except Exception` catches in `run_forever`.

### What `AgentRunner` does NOT do (deferred)

- Streaming (v0.7).
- Mid-turn message injection (v0.10).
- Checkpoints (v0.9).
- Context governance / snip / orphan-tool cleanup (v0.8).
- Empty-retry, length-recovery (later).

```python
class AgentLoop:
    def __init__(
        self,
        bus: MessageBus,
        provider: LLMProvider,
        tools: dict[str, ToolLike],

    ) -> None:
        self._bus = bus
        self._provider = provider
        self._tools = tools

    async def run_forever(self) -> None: ...                       # unchanged
```

Constructor signature unchanged from v0.2 except added optional `max_iterations`. `__main__.py` keeps working without edits.

## 6. Test strategy

### `tests/agent/test_runner.py` (new)

Uses `ScriptedProvider` to drive the runner with canned responses. Coverage:

- Single text response → `stop_reason="completed"`, no tools used.
- One tool call → tool executes, result fed back, final text returned, `tools_used == ["add"]`.
- Multiple tool calls in one response → all execute, results appended in order.
- Unknown tool → error string in result, run continues.
- Tool raises → error string, run continues.
- `max_iterations` reached → `stop_reason="max_iterations"`, `final_content=None`.

### `tests/agent/test_loop.py` (rewritten)

Uses `ScriptedRunner` (new test helper) to test FSM-level behaviour:

- BUILD seeds messages and tool schemas correctly.
- RUN delegates to runner and stores result.
- RESPOND publishes `OutboundMessage` with the right channel/chat_id/content.
- Empty content (runner returned `None`) → publishes empty string (matches v0.2 behaviour).
- `run_forever` loops over inbound messages.

The 4 existing "tool call" tests in v0.2's test_loop.py logically move to `test_runner.py` — the loop no longer executes tools, the runner does.

### `tests/_helpers/scripted_runner.py` (new)

```python
class ScriptedRunner:
    def __init__(self, results: list[AgentRunResult]) -> None: ...
    async def run(self, spec: AgentRunSpec) -> AgentRunResult: ...
```

Plays back a fixed list of `AgentRunResult`s. Raises on exhaustion.

### `tests/test_e2e.py`

Unchanged behaviour, but now exercises the FSM path. Still uses `ScriptedProvider`. Still hermetic.

## 7. Python concepts taught in v0.3

- **`enum.Enum`** with `auto()` for state-machine sentinel values.
- **`match/case`** structural pattern matching — Python 3.10+ feature, perfect fit for FSMs.
- **Separating orchestration from execution** — AgentLoop = "what happens around the LLM turn", AgentRunner = "the LLM turn itself".
- **Dataclass-based input/output contracts** (`AgentRunSpec`/`AgentRunResult`) — well-defined interfaces between modules.
- **Mutable context object** (`_TurnContext`) passed between handler methods — keeps handler signatures uniform without globals.
- **Test doubles at different layers** — `ScriptedProvider` for runner tests, `ScriptedRunner` for loop tests. Each layer tested against the interface of the layer below.
