## Real API call (verified)

User created a `.env` file (gitignored) holding `ANTHROPIC_API_KEY=...`.
Loaded via `set -a; . .env; set +a`. Then a single user message was
piped into `python -m peppermill` and the process was killed after
12s.

```
$ set -a; . .env; set +a
$ echo "What is 2 + 3? Use the add tool." | python -m peppermill
PepperMill v0.2 — type a message and press Enter. Ctrl-D to exit.
INFO httpx: HTTP Request: POST https://api.anthropic.com/v1/messages "HTTP/1.1 200 OK"
INFO httpx: HTTP Request: POST https://api.anthropic.com/v1/messages "HTTP/1.1 200 OK"

The answer is **5**. When you add 2 and 3 together, you get 5.
```

Two round-trips to api.anthropic.com:
1. First call: provider returned a `tool_use` block requesting `add(2, 3)`.
2. AgentRunner executed `AddTool.execute(a=2, b=3) → 5`, fed the result back.
3. Second call: provider returned the final text response.

The FSM (`BUILD → RUN → RESPOND → DONE`) is **transparent** — same observable
behaviour as v0.2 with the new internal architecture.

(The welcome string still reads "v0.2" — left intentionally for v0.3; v1.0
will sweep the banner. v0.3 explicitly does not touch `__main__.py`.)

## Behaviour expected end-to-end

1. CLIChannel reads user input.
2. AgentLoop.run_once is called; FSM enters BUILD.
3. _state_build seeds `initial_messages` + `tool_schemas`.
4. FSM enters RUN; AgentLoop constructs an AgentRunSpec and delegates to AgentRunner.
5. AgentRunner.run loops over provider.chat() + tool execution.
6. AgentRunner returns AgentRunResult.
7. FSM enters RESPOND; OutboundMessage is published.
8. CLIChannel prints to stdout.
9. FSM enters DONE; run_once returns.

## Observations / open issues

- AgentLoop's public constructor signature is backward-compatible with v0.2
  (`max_iterations` is the only new kwarg, and it defaults to 20).
- No production-code change to `__main__.py`, `peppermill/providers/*`,
  `peppermill/channels/*`, `peppermill/bus/*`. v0.3 was purely an agent/
  refactor.
- Tools no longer execute inside `AgentLoop` — they execute in `AgentRunner`.
  Existing tool tests stay in `tests/agent/tools/`.
- The graceful-shutdown gap from v0.2 still exists (Ctrl-D kills the reader
  task but `run_forever` keeps awaiting the queue). v0.10 fixes lifecycle.
